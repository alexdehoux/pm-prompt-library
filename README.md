## How to Use

Each folder corresponds to a PM workflow area. Inside, you'll find individual prompt files organized by use case.

Every prompt follows this format:

```
**Prompt Name**
A short title describing what the prompt does.

**When to use**
The situation or trigger for using this prompt.

**Prompt**
The actual prompt text. Variables are written in [BRACKETS] — replace them with your own context before use.

**Tips** *(optional)*
Notes on how to get better results, common variations, or follow-up prompts.
```

To use a prompt, copy it into your AI tool of choice, fill in the bracketed variables, and adjust the output as needed. Most prompts are written for general-purpose LLMs (ChatGPT, Claude, Gemini) and work without special configuration.

## Contributing

Contributions are welcome. If you have a prompt that's saved you time or improved your output quality, feel free to add it.

**To contribute:**

1. Fork this repo
2. Add your prompt to the relevant folder using the format described above
3. If your use case doesn't fit an existing folder, propose a new one in your PR description
4. Submit a pull request with a brief note on what the prompt does and when you use it

Please keep prompts practical and grounded in real PM work. Avoid prompts that are too generic to be useful or too narrow to apply beyond a single context.
